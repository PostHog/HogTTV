HogTTV Privacy Policy

Last updated: May 19, 2025

HogTTV is a Chrome extension that displays your Slack workspace's custom emojis inside Google Meet chat. This policy explains what data the extension handles and how.

What data we collect

The extension stores the following data locally on your device using Chrome's built-in storage API:

  • Slack OAuth token — obtained when you connect your workspace. Used to fetch your custom emoji list from the Slack API. Never sent to any server other than Slack.
  • Workspace name — displayed in the extension popup so you know which workspace is connected.
  • Emoji cache — a local copy of your workspace's emoji names and image URLs, refreshed every 4 hours. Stored only on your device.

We do not collect names, email addresses, message content, browsing history, or any other personal information.

OAuth server

Connecting to Slack requires exchanging an authorization code for a token, which cannot be done safely inside a browser extension (it would expose the client secret). To handle this, HogTTV uses a minimal server at hogttv-server.vercel.app as a stateless pass-through: it receives the authorization code from Slack, exchanges it for a token, and immediately redirects that token back to your extension. The server does not log, store, or transmit any user data.

Third-party services

The extension communicates with:

  • Slack API (slack.com/api/emoji.list) — to fetch your workspace's custom emojis.
  • Slack OAuth (slack.com/oauth/v2/authorize) — to authenticate you.
  • HogTTV OAuth server (hogttv-server.vercel.app) — solely to complete the OAuth code exchange described above.

No data is sold or shared with any other third party.

Data retention and deletion

All data is stored locally on your device. You can delete it at any time by clicking Disconnect in the extension popup, which clears all stored tokens and cached emojis. Uninstalling the extension removes all local data automatically.
